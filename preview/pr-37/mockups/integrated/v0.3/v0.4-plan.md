# v0.4 Implementation Plan — Full Game Flow

## Goal
Complete the full game experience: Start Screen → Battle Forge → Game (6 phases) → Game End.
With animated transitions between screens. Music continuity across states.

## Architecture: Screen Router

The integrated prototype becomes a full SPA with screen routing:

```
START SCREEN → BATTLE FORGE → GAME (Deploy→Move→Shoot→Charge→Fight→Game End)
     ↑                                                              |
     └──────────────────── PLAY AGAIN ──────────────────────────────┘
```

### Screen States
1. **start** — Cinematic splash screen (from start-game mockup)
2. **battle-forge** — Army selection (from battle-forge v0.8)
3. **game** — The existing 6-phase integrated gameplay
4. **game-end** — Already built (overlay on game screen)

### Implementation Approach
Instead of iframing standalone mockups, we:
1. Create a `<div id="screen-root">` that contains all screens
2. Each screen is a `<div class="screen">` that gets shown/hidden
3. The game screen is the existing `<div id="app">` with roster + battlefield
4. Start and Battle Forge screens are new divs with their HTML inlined
5. CSS transitions between screens (fade out → fade in)

### Music Continuity
- Ambient audio element persists across all screens
- Start screen: ambient-loop.mp3 starts on first click
- Battle Forge: same audio continues
- Game: audio continues (or could swap to battle music)
- Game End: audio continues
